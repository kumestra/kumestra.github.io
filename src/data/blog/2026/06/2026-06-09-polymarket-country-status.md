## 先说结论

如果把“Polymarket 友好”定义为“可以从当地正常打开并开新仓”，判断方法其实很直接：看该国家或地区是否出现在 Polymarket 官方地理限制列表里。没有出现在“Blocked”“Close-only”“Frontend UI restricted”或特定地区封锁列表里的地方，通常可以视为国际站相对友好。

本文关注的是 `polymarket.com` 国际站，不是面向美国用户的 `polymarket.us`。截至 2026-06-09，Polymarket 官方文档显示，美国在国际站仍是受限地区；美国用户应区分 Polymarket International 和 Polymarket US。

这不是法律意见。一个地方没有被 Polymarket 官方 geoblock 拦住，不等于当地监管一定明确允许。实际交易前还要看本地法律、网络访问情况，以及 Polymarket 的实时风控。

## 状态分类

| 状态 | 含义 |
|---|---|
| 可开新仓 | 没有出现在官方限制列表中，通常可以正常交易 |
| 仅平仓 | 可以关闭已有仓位，但不能开新仓 |
| 完全受限 | 不能正常访问或不能下单 |
| 前端限制 | 正常网页前端受限，API 文档中可能还有更细的技术说明 |
| 地区限制 | 国家整体未必受限，但某些省、州或地区受限 |

Polymarket 的限制主要按“物理位置”判断，而不是只看国籍或居住地。官方也明确禁止使用 VPN 等工具绕过地理限制。

## 仅平仓地区

这些地方不是完全无法处理已有仓位，但不能开新仓：

| 国家或地区 | 代码 | 状态 |
|---|---|---|
| 新加坡 | SG | 仅平仓 |
| 波兰 | PL | 仅平仓 |
| 泰国 | TH | 仅平仓 |
| 台湾 | TW | 仅平仓 |

这类状态对已有仓位的人尤其重要：它通常意味着可以退出或结算，但不能继续加仓或开新市场。

## 日本的特殊情况

日本不是“仅平仓”。Polymarket API 文档把 `JP | Japan` 标为 `Frontend UI restricted`，也就是正常网页前端受限；帮助中心页面则把日本放进完全受限列表。对普通用户来说，结论可以简单理解为：日本不是一个适合正常使用 Polymarket 国际站开仓的地区。

## 国家内部的地区限制

有些国家不是全国封锁，而是特定地区受限：

| 国家 | 受限地区 | 代码 |
|---|---|---|
| 加拿大 | 安大略省 | ON |
| 乌克兰 | 克里米亚 | 43 |
| 乌克兰 | 顿涅茨克 | 14 |
| 乌克兰 | 卢甘斯克 | 09 |

所以加拿大不能简单说“全境友好”。如果人在多伦多或安大略省其他地方，官方列表显示交易服务不可用；如果人在加拿大其他省份，则需要按实时检测结果判断。

## 亚洲几个常见问题

香港和韩国是最常被问到的两个地区。按当前官方列表看，香港没有出现在限制名单中，韩国也没有出现在限制名单中。相反，台湾和新加坡都是仅平仓；日本前端受限；朝鲜完全受限；缅甸也完全受限。

中国大陆没有出现在这份官方 API 限制表中，但这只说明 Polymarket 的官方 geoblock 表没有把 `CN` 列为受限。实际访问情况、支付链路、当地法律和平台风控仍然需要单独判断。

## 实用判断流程

可以用下面这个流程判断一个地方是否“Polymarket 友好”：

1. 先看是否在完全受限列表中。
2. 再看是否是仅平仓地区。
3. 再看是否有省、州、地区级别的限制。
4. 如果都没有，再用 Polymarket 的实时 geoblock 接口或网站提示确认。
5. 最后再看当地法律和自己的账户合规要求。

对普通用户来说，最重要的是不要把“能打开网页”误认为“可以合法、稳定地交易”。真正关键的是能否开新仓、能否入金出金、能否在账户风控下长期正常使用。
